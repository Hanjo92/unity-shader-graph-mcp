# 1.1.0 Plan

`1.0.0` shipped the first stable package-backed editing engine plus the live MCP
transport and Unity batchmode bridge. `1.1.0` should carry the remaining
post-`1.0.0` work forward without reopening broad "all nodes / all ports"
scope.

## Outcome

`1.1.0` should mean:

- the stable `1.0.0` base grows into a wider but still explicit supported matrix
- practical texture, UV, normal, and follow-up utility workflows cover more real authoring cases
- broader node and property support can be promoted without drifting from the verified package-backed surface
- graph and subgraph authoring stay contract-compatible with the shipped `1.0.0` flow

## Proposed Scope

### 1. Remaining Connection Matrix Expansion

- widen the remaining package-backed routing beyond the current `1.0.0` matrix
- keep expanding texture, UV, sample, normal, and utility workflows one verified family at a time
- extend boolean, scalar, vector, and color routing only where the package-backed path stays stable
- keep `list_supported_connections`, response metadata, and smoke coverage aligned with the real runtime surface

### 2. Broader Authoring Surface

- promote more blackboard property kinds beyond the current stable `Color` and `Float`/`Vector1` base
- promote more graph-addable node types from the discovered catalog into the verified runtime catalog
- keep diagnostics broader than runtime support only when that distinction stays explicit in docs and metadata

### 3. Subgraph Composition Follow-Through

- build on the current subgraph lifecycle and summary support with the next safe composition paths
- harden graph-contract import/export around graph vs subgraph boundaries
- keep graph-only and subgraph-only connection rules visible to external callers

### 4. Expansion Hardening

- keep sample fixtures and regression coverage growing with every new verified path
- preserve clear unsupported-path validation so the boundary remains easy to understand
- keep the compatibility report and matrix current whenever the promoted runtime surface changes

## Planning Rule

Prefer extending the existing `shadergraph_asset` contract and diagnostics rather
than introducing new tool shapes unless a real separation pressure appears.

## Recommended Execution Order

1. refresh the `1.1.0` backlog from the currently unsupported but desired workflows
2. land remaining connection-matrix paths in small verified slices
3. promote broader node and property support only with matching smoke and metadata tests
4. harden subgraph composition and contract replay on top of the widened base

## Acceptance Criteria

- one additional workflow family beyond the `1.0.0` base is verified end-to-end through the real package-backed path
- `list_supported_connections` and the runtime response metadata stay in sync with the supported matrix
- promoted node and property coverage is reflected in summaries, catalogs, and diagnostics
- graph and subgraph contract replay rules stay explicit and test-locked
- docs describe the widened `1.1.0` boundary without promising universal support

## Explicit Non-Goals

- universal texture, UV, and normal workflow coverage in one cut
- custom-function authoring
- universal node and port support
- template-backed graph creation beyond the current blank path
- broad cross-version support promises beyond the compatibility-matrix discipline

Those stay better suited for `1.2.x` or later.
