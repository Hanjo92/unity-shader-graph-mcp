# 1.1.0 Plan

`1.0.0` froze the first stable package-backed editing engine plus the live MCP
transport and Unity batchmode bridge. `1.1.0` should improve authoring
ergonomics without reopening broad "all nodes / all ports" scope.

## Outcome

`1.1.0` should mean:

- external callers can find the right node more easily
- common graph edits do not require brittle manual object-id bookkeeping
- graph layout can be made deterministic enough for automation
- the contract grows in a way that stays compatible with the current `1.0.0` paths

## Proposed Scope

### 1. Query Ergonomics

- node lookup by exact `objectId`
- node lookup by display title when it is unique
- node lookup by canonical node type or alias
- explicit supported-node catalog query for external callers

### 2. Mutation Ergonomics

- update an existing blackboard property instead of only adding one
- move a node to an explicit position
- place a new node relative to an existing node or a small layout preset

### 3. First Subgraph Path

- read subgraph summary
- create blank subgraph if the package surface is low-risk
- defer deep subgraph composition until the base query/mutation ergonomics are stable

## Suggested Contract Additions

Candidate new actions:

- `find_node`
- `move_node`
- `update_property`
- `list_supported_nodes`
- optional: `read_subgraph_summary`
- optional: `create_subgraph`

These should remain inside the current `shadergraph_asset` tool unless a clear
separation pressure appears.

## Recommended Execution Order

1. `find_node` plus supported-node catalog query
2. `update_property`
3. `move_node` and deterministic layout helpers
4. low-risk subgraph read/create path

## Acceptance Criteria

- one external MCP flow can locate a node without hard-coded manual inspection
- one external MCP flow can update an existing property in place
- one external MCP flow can place or move nodes deterministically
- EditMode tests lock the new query and mutation paths
- docs show one recommended ergonomic authoring flow on top of the `1.0.0` base

## Explicit Non-Goals

- universal texture and UV workflow coverage
- custom-function authoring
- universal node and port support
- template-backed graph creation beyond the current blank path

Those stay better suited for `1.2.x` or later.
